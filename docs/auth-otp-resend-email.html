Auth email · the 500 fix

Send the OTP through Resend, not Supabase.

Your login code request is hitting a 500 because Supabase's built-in email service can't deliver it. You already run Resend — here's how to route auth emails through it and kill the error.

What the 500 actually means

The call to /auth/v1/otp reached Supabase fine — the OTP was minted. Supabase then returned “Error sending confirmation email.” The default email service is dev-only: it delivers only to project team members and is capped at ~2–3/hour. Sending a code to a normal Gmail through it fails. Swap in Resend and it's solved.

The mental model

Supabase keeps the OTP. Resend just carries the email.

You can't move OTP logic off Supabase — but you can absolutely move the email delivery. That's the part that's broken.

Stays on Supabase

Minting & verifying the code

signInWithOtp() generates the 6-digit code; verifyOtp() checks it and starts the session. This is core auth — it can't live in Resend.

Moves to Resend

Delivering the email

The actual “here's your code” message. Today Supabase's default sender chokes on it. Resend delivers to any address, with a higher rate limit.

Pick a path

Two ways to send through Resend.

A

Resend as custom SMTP

Config only · ~5 minutes · zero code

Recommended

Supabase still orchestrates the OTP, but hands the email to Resend's SMTP relay. This directly fixes the 500 and bumps the send limit (~2 → ~30/hr).

B

Send Email Auth Hook

Full control · reuses your lib/email/resend.ts · more setup

Advanced

Supabase mints the OTP, then calls your endpoint with the code instead of sending anything itself. You send it via the Resend API — same templates, branding and seam as your transactional emails. Pick this when you want auth mail to match the rest pixel-for-pixel.

  1. Build a hook endpoint

    e.g. src/app/api/auth/send-email/route.ts — receives the payload, verifies the webhook secret, renders the code, sends with your Resend provider.

  2. Register the hook

    Supabase → Authentication → Hooks → Send Email → point it at your deployed URL + secret.

  3. Supabase stops sending email

    Every auth email now flows through your code path — OTP, magic link, recovery — all on Resend.

// sketch — src/app/api/auth/send-email/route.ts
export async function POST(req) {
  const { user, email_data } = verifyHook(req); // check the secret
  await email.send({
    to: user.email,
    subject: `Your code: ${email_data.token}`,
    html: renderOtp(email_data.token),   // your branded template
  });
  return Response.json({});
}
