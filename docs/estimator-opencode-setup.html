The estimator talks to any OpenAI-compatible gateway. Point three env vars at opencode Zen, drop in your key, and every request gets web-grounded shipping & item pricing.

How a price is made

Four steps, fully fenced.

1

Classify the item

A deterministic keyword pass tags the request — trading_card, sneakers, apparel, watch, camera, video_game, figure — no model call.

2

Pull live comps from trusted sites (Exa)

Searches that category's trusted domains only and returns recent listings. Optional — add EXA_API_KEY to switch it on; skipped silently if absent or slow (10s cap).

3

Reason over them (opencode / DeepSeek)

The model gets the category skill + live comps and returns JSON: price, low/high, confidence, carrier, and which sources it used. Gateway-agnostic with JSON-mode retry + tolerant parsing.

4

Clamp, cap & fall back

Figures are clamped to safe bounds and never exceed the budget cap. Any failure falls back to the deterministic stub, so checkout never blocks. Memoized 10 min so a view + deposit don't double-bill.

Setup

Paste your opencode key, restart.

.env.local 
ESTIMATOR_PROVIDER=deepseek
DEEPSEEK_BASE_URL=https://opencode.ai/zen/v1
DEEPSEEK_API_KEY=<your opencode key>
DEEPSEEK_MODEL=deepseek-v4-flash   # or deepseek-v4-pro

# optional — live web comps
EXA_API_KEY=<your exa key>

Other gateways, same vars

It's just “base URL + key + model.” Swap in DeepSeek-direct (api.deepseek.com/v1) or OpenRouter (openrouter.ai/api/v1) with zero code change.

Then mirror in Vercel

Settings → Environment Variables → add the same four (or five) keys → redeploy. .env.local is local only.

Read before production

Two honest cautions on opencode Go.

Intended use / terms

opencode Go is a flat-fee plan for the coding agent, not a production app backend. Powering every checkout's pricing call may breach their terms and risks the key being throttled or revoked. For a live customer feature, prefer per-token opencode Zen, DeepSeek-direct, or OpenRouter.

Reliability

A coding-agent gateway isn't SLA'd for checkout traffic. If it's slow or down, the estimator silently uses the flat stub (¥4,000) — safe, but not the smart pricing you want.

Recommended path

Use opencode Go now to verify end-to-end (cheap, instant). For production, switch the DEEPSEEK_* values to a dedicated inference key — zero code change, just different env.
