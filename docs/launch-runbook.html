Create a Resend account & verify your sending domain

At resend.com: add your domain, then add the DNS records they give you (SPF, DKIM, and a return-path) at your DNS host. Verification can take a few minutes to an hour. A verified domain is required to send from @yourdomain.com.

Create an API key

Resend → API Keys → Create (Sending access is enough). Copy the re_… value — you only see it once.

Set the email env vars

Locally in .env.local, and in Vercel → Project → Settings → Environment Variables (Production):

EMAIL_PROVIDER=resend
RESEND_API_KEY=re_…
EMAIL_FROM=Finders Keepers <hello@yourdomain.com>   # must be on the verified domain
◆Leaving EMAIL_PROVIDER=stub (the default) is safe — it just logs emails to the server console and sends nothing. That's how dev works with no key.

Go live: register the production webhook & swap to live keys

In Stripe (Live mode) → Developers → Webhooks → Add endpoint:

URL:    https://your-domain.com/api/stripe/webhook
Events: checkout.session.completed,
        payment_intent.payment_failed,
        charge.refunded

Copy that endpoint's live whsec_…, then set the live values (sk_live_…, pk_live_…, live whsec_…) in Vercel Production env vars. Redeploy.

3

Supabase Auth email (signup confirmation)

New email/password signups need a confirmation email to be delivered, or accounts can't verify. This is separate from the Resend transactional email in step 1 — it's Supabase's own auth mail.

Supabase dashboard → Authentication

Decide: confirm-on-signup or instant sign-in

Authentication → Providers → Email. For real users, keep “Confirm email” ON (recommended). If you'd rather skip confirmation, turn it off — but then anyone can sign up with any address.

Configure a custom SMTP sender (recommended for production)

Authentication → Emails → SMTP Settings. Supabase's built-in email is rate-limited and not meant for production volume. Point it at Resend's SMTP (or another provider) so confirmation mail actually lands. Set the same From domain you verified in step 1.

Set the redirect URLs

Authentication → URL Configuration. Add your production Site URL and allow https://your-domain.com/auth/callback as a redirect — that's the route the app uses after confirmation/OAuth.

(Optional) Apple sign-in

It's intentionally hidden in production until configured. To enable: add Apple as an OAuth provider in Supabase Auth, then set NEXT_PUBLIC_APPLE_AUTH_ENABLED=true in Vercel. Email auth works without this.

✓Done when: a brand-new signup receives a confirmation email and can verify, landing back on the app via /auth/callback.

Error visibility — the free path

When a payment, webhook, or release silently fails in production, you want to find out fast — not from a confused customer three days later. No paid tool needed: the app already logs every money path to the console, and Vercel surfaces those for free.

free · Vercel Logs + Observability · optional free-tier log drain

✓The code already logs the risky paths — [notify] …, [email:resend] …, escrow/webhook failures. Steps below just make those visible and alertable for $0.
Confirm Vercel Runtime Logs (built-in, free)

Vercel → your project → Logs. This streams server/function output, including the app's console.error from the webhook and lifecycle operations. After a live transaction, check there are no unexpected errors around /api/stripe/webhook.

Turn on Vercel Observability

Vercel → project → Observability. The free tier charts function error rates and lets you spot a spike on the webhook or checkout routes without any code change.

(Optional) Add a free-tier log drain with alerts

Runtime logs are ephemeral. For retained logs + email/Slack alerts at $0, connect a free-tier drain — e.g. Better Stack or Axiom — via Vercel → project → Integrations. Set an alert on log lines matching [notify], [email:resend], or webhook errors.

Verify the money paths are observable

Trigger a deliberate failure (e.g. a Stripe test event the handler rejects) and confirm it shows up in Vercel Logs / your drain. That's your safety net for real-money issues.

◆If you later want richer error grouping for free, Sentry's own free tier and the open-source GlitchTip are drop-in options — but Vercel's built-ins are enough to launch.
✓Done when: a deliberately triggered error is visible in Vercel Logs (and fires your drain alert, if you set one up).
