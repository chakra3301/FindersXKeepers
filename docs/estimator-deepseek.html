New capability

A DeepSeek estimator
for shipping & escrow pricing.

Before we source an item we now estimate the Japan → customer shipping and the item's market value — sizing the displayed quote and the escrow hold. Wired behind a one-file seam, with a deterministic fallback so checkout is never blocked.

No new dependencies 101 tests green Four pricing lines intact

The idea

One env flag swaps a flat guess for a real model.

Shipping estimate

From item, condition & destination country, DeepSeek estimates all-in tracked international shipping in yen.

Item-value estimate

A second-hand market price (with a low/high range) to size the escrow hold — clamped to never exceed the budget cap.

Never blocks checkout

Any network, parse or timeout failure silently falls back to the deterministic stub, tagged fallback.

How a quote is built

From request to hold — same number, every time.

1

Customer opens checkout

The server computes the shipping estimate once via estimator.estimateShipping() and hands it to the form.

2

Four-line quote renders

Item cost · finder's fee · shipping · tax — never collapsed. Rush re-prices the fee only, so the shipping line stays stable.

3

Escrow hold is sized

depositForRequest() asks the same memoized estimator, so the displayed estimate and the hold are built from one input.

4

Order locks & reconciles

On approval the real four-line order is written; the model figure is only the pre-sourcing estimate, reconciled as before.

The five non-negotiables stay intact

Pricing is still four separate lines. Funds still flow only through the processor — the estimator only feeds inputs to computeQuote, it never holds money or collapses the breakdown. The item-value estimate can never authorise more than the customer's cap.

Turn it on

Two lines in .env.local.

# Route shipping + item-value estimates through DeepSeek
ESTIMATOR_PROVIDER=deepseek
DEEPSEEK_API_KEY=sk-...

# Optional — these are the defaults
# DEEPSEEK_BASE_URL=https://api.deepseek.com/v1
# DEEPSEEK_MODEL=deepseek-chat

Leave it unset (or =stub) and everything runs on the deterministic estimator — no key, no network. Same way escrow and email already work.
